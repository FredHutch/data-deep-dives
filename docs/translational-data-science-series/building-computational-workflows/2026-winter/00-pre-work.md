# Pre-Work: Building Computational Workflows
## **Data Deep Dive | March 11, 2026 | 12-2pm**

As a part of this Deep Dive we will be demonstrating how to modify a workflow and how to submit one. If you would like to follow along you will need to finish the setup below. This ensures you can fully participate without spending session time troubleshooting access.

> Estimated time: 10–15 minutes

## 1️⃣ Confirm Cluster Access

- Ensure you have cluster credentials set up with SciComp ([go here](https://sciwiki.fredhutch.org/scicomputing/access_credentials/) for more details)
- Test that you can log in by going to the terminal and typing:

  `ssh <your_hutch_username>@rhino.fredhutch.org`

  ([more details here](https://sciwiki.fredhutch.org/scicomputing/access_credentials/#testing-your-credentials) on how to test your credentials)
- If you do not have access, [contact SciComp](https://scicomp-self-service.fredhutch.org/) before the session.


## 2️⃣ Log into PROOF Workbench Once

- Visit the [PROOF Workbench interface](https://pwb.fredhutch.org/)
- Log in using your Hutch credentials (go [here](https://sciwiki.fredhutch.org/datademos/pwb-tutorial/) to read more about PROOF Workbench)
- Confirm you can access the dashboard
- (Optional) Start a Server (go [here](https://sciwiki.fredhutch.org/datademos/pwb-tutorial/#starting-your-first-proof-server) for instructions)
- You do not need to submit anything yet — just verify access.


## 3️⃣ Download the Demo Workflows (Optional but Helpful)

- We will use a small example WDL during the session.
- Download the demo WDLs (download all the files inside the wdls folder [here](https://github.com/FredHutch/data-deep-dives/tree/main/docs/translational-data-science-series/building-computational-workflows/2026-winter/wdls))

## Optional (Helpful Background)

- If you want to arrive with extra context start by reading about [Computational Workflows](https://sciwiki.fredhutch.org/datascience/using_workflows/) and the [Workflow Description Language](https://sciwiki.fredhutch.org/datascience/wdl_workflows/)
- Skim the minimal WDL example shared in advance

## 🆘 If You Get Stuck

- Do not debug alone.
- Post in Slack: #workflow-managers
- Email: datascience@fredhutch.org

> We would much rather help you beforehand than during the session.
